Prototype — supports contract review and does not replace professional legal or privacy assessment.

The Problem

Data-processing contract → Manual privacy review → Slow · inconsistent · hard to trace

Time-consuming review

Privacy teams read long contracts and compare many clauses against regulatory requirements — by hand.

Missing or unclear clauses

Purpose, retention, deletion, security, and data-transfer terms may be absent or ambiguous.

Inconsistent decisions

Different reviewers read the same clause differently — results are hard to standardize and trace.

Organizations need a faster, more consistent way to identify contract risks — while keeping the final decision under human oversight.

What Makes It Agentic?

Shared State — remembers what happened between steps Specialized Roles — separate analyze & review Conditional Routing — next step depends on the result Revision Loop — feedback re-fed, bounded by MAX_RETRIES

Human-review note: high-risk or uncertain findings are flagged for human review, but the current implementation does not yet pause and resume the graph for a reviewer decision.

System Architecture

LangGraph orchestrator
↺ Retry if score below threshold
Interface Service Security Agent node Storage Observability

Current implementation: built in capstone_contract_audit_v2.py (FastAPI · LangGraph · pypdf · ChromaDB · MinIO · PostgreSQL · Prometheus), shipped with docker-compose.yml. A MOCK=1 mode runs the full flow offline for the demo.
